# OmniFuse Agent Rules

## Language

This project is maintained in English.

- Write all project-facing artifacts in English: documentation, plans, changelogs, release notes, PR descriptions, issue text, code comments, test fixtures, and commit messages.
- Keep code identifiers, API fields, commands, paths, crate names, and product names in their canonical form.
- Do not localize repository artifacts to the user's chat language. If the conversation is in another language, respond to the user in that language but keep repository changes in English.
- If an existing artifact is not in English, translate it before making substantial edits to it.

## Development

- Follow [CONTRIBUTING.md](CONTRIBUTING.md) for architecture, testing, linting, and release workflow.
- Prefer small, consistent changes over broad rewrites.
- Preserve existing module boundaries unless the task explicitly asks for architectural work.
